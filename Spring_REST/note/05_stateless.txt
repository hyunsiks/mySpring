# HTTP 프로토콜은 비연결지향형(Connecitonless) 프로토콜이다

	- 서버는 클라이언트를 기억하지 못한다
	- 이전에 클라이언트가 무엇을 했는지 기억하기 위해서는 다양한 편법을 사용해야한다
	- 즉, 클라이언트의 상태가 서버에 저장되지 않는다 (Stateless)
	
# 세션

	- 각 클라이언트에 발급한 세션ID를 전달받아 서버에서 키값으로 활용한다

# 쿠키

	- 서버가 클라이언트 쪽에 저장해 둘 수 있는 데이터
	- 응답과 함께 실어보낼 수 있다
	- 클라이언트 쪽에서 위/변조가 자유롭기 때문에 보안에 주의해야한다
	- 클라이언트 쪽에 저장된 쿠키는 해당 클라이언트가 보내는 모든 요청에 포함된다
	  (쿠기가 적용될 URL범위에 따라 전달되지 않을 수도 있다)
	  
#쿠키의 필드값들

	- MaxAge : 쿠키가 만료되는 시간을 초 단위로 설정한다. 기본값은 -1이고, 브라우저 종료시 함께 샂게된다
	           (오늘 하루동안 더 이상 보지 않음 가능)
	- path : path에 설정해놓은 경로의 하위 경로에서 모두 적용되는 쿠키로 설정한다
	- domain : domain에 설정해놓은 IP주소(도메인이름)에서 모두 적용되는 쿠키로 설정한다
	- HttpOnly : 쿠키에 자바스크립트 문법을 적용하여 나쁜짓을 하는 것을 예방한다.
