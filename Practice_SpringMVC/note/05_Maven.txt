# POM (Project Object Model)
	- 프로젝트에 대한 정보를 담고 있는 객체 모델
	- Maven은 프로젝트를 항상 pom.xml에 적혀있는대로 유지해주는 프로그램이다
	- pom.xml에는 프로젝트의 모든 설정과 의존성에 대한 정보들을 담고 있다
	
# pom.xml의 태그 정보
	<name> : 프로젝트 이름
	<artifactID> : 외부에서 볼 우리 프로젝트 이름
	<groupID> : 프로젝트의 그룹 ID
	<version> : 버전 설정
	<packaging> : 배포할 파일의 타입을 설정. 서버 종류에 따라 다른 확장자로 내보낸다.(톰캣의 경우에 war로 되어있다)
	<dependencies> : 이 프로젝트에서 의존하는 다른 프로젝트들을 적는 곳
	<dependency> : 의존하는 프로젝트의 POM 정보를 적는다
				   <groupID><artifactID><version><scope> 등을 적는다
	
# <scope>
	- 해당 <dependency>가 프로젝트에서 영향을 미치는 범위를 설정한다
	- 설정하지 않을 시 default값으로 
	- compile : 컴파일 할 때 필요한 의존성
				컴파일 당시에도 사용되고, 배포에도 포함된다(기본값, scope설정 안된 모든것)
	- runtime : 런타임에 필요한 의존성(실행할때만 필요한것)
				컴파일 당시에는 사용되지 않고, 배포에는 포함된다(ex. ojdbc, sl4j 등)
	- provided : 컴파일 할 때는 필요하지만 런타임 때에는 기본적으로 제공되는 의존성
				 컴파일 당시에는 사용되고, 배포에는 포함되지 않는다.(ex. jsp 등)
	- test : 테스트 코드 컴파일 진행시에만 필요한 의존성
			 테스트시에만 사용되고, 배포에는 포함되지 않는다
